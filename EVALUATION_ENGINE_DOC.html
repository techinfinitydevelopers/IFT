1. Overview

The IFT (India Future Tycoon) Evaluation Engine is an AI-powered system that evaluates student innovation idea submissions. It uses a hybrid AI approach:

  • Claude 3.5 Sonnet (via OpenRouter) — Evaluates text answers, runs coherence checks, analyzes images, and checks document relevance.
  • Gemini 2.0 Flash (via OpenRouter) — Natively analyzes video files for content and relevance.

Each submission is scored on a 0-100 scale (10 parameters × 0-10 marks each), with coherence penalties (-1 per failed check) and attachment penalties (up to -5). The system is designed to be strict and rigorous — only truly strong ideas score high.

Key Change (v3): The system now uses 12 questions (Q1-Q12), a 0-10 scoring scale per parameter (max 100), 10 coherence cross-checks with penalty mapping, and parallel execution of evaluation, coherence, and attachment analysis for faster results.

Students can also upload 3 types of attachments: image, document, and video.

3. Evaluation Flow (Parallel Architecture)

When a submission is evaluated, the engine runs 3 major tasks in parallel using a ThreadPoolExecutor for maximum speed:

                    +---> [Main AI Evaluation] ---> 10 parameter scores (0-10 each)
                    |         (Claude 3.5 Sonnet)
                    |
Submission Data --->+---> [Coherence Checks]   ---> 10 cross-check results (pass/fail)
                    |         (Claude 3.5 Sonnet, separate client)
                    |
                    +---> [Attachment Analysis]  ---> file relevance + mismatch info
                              (Claude + Gemini, separate client)

                              === All 3 run simultaneously ===

Then sequentially:
  1. Parse evaluation scores
  2. Apply coherence penalties (-1 per failed check to affected parameters)
  3. If >5 coherence failures: DISQUALIFY (all scores = 0)
  4. Process attachment mismatch penalties
  5. Create AIEvaluation record with all data in one DB write
  6. Update submission status to 'evaluated'
Performance: Before parallelization, evaluation took 40-65+ seconds (sequential API calls). Now it takes max(eval, coherence, attachments) — typically 20-35 seconds, a 40-60% improvement.

4. Scoring System - 10 Parameters (0-10 Scale)

The evaluation uses exactly 10 parameters divided into two categories. Each parameter is scored on a 0-10 scale (0 = worst, 10 = best).

Idea Parameters (5 parameters)

2Ease of ImplementationCan this idea be built?Clear step-by-step plan. Resources available and listed. Team has expertise.No clear plan. Resources unavailable. No expertise.
3FeasibilityIs the plan realistic?Resources realistically available. Practical plan with milestones. Challenges acknowledged with mitigation.Plan seems unrealistic. Resources unavailable. No awareness of constraints.
4ImpactHow many people benefit?Widespread customer base. Solution critical for users. Removes pains AND adds new benefits. Shows scale.Sporadic users, no visible positive difference

Team Parameters (5 parameters)

Since this is a text-based submission (not face-to-face), team qualities are inferred from the writing:

#ParameterWhat It MeasuresScore 8-10 (Best)Score 0-3 (Worst)
6Conceptual ClarityDoes the team understand their own idea?Clear on idea AND execution. Knows non-negotiable vs nice-to-have features. Problem-solution link crystal clear.Still unclear about the idea itself. No execution plan.
7EmpathyDoes the team understand user pain?Deep empathy — put themselves in users' shoes. Describes real user stories. Removes pains AND brings extra gains.Focused on thrill of ideation, ignored user needs
8CreativityIs the approach innovative?Divergent/out-of-box thinking. Trend-setter. Unexpected element clearly described.Average approach. No creative temperament. Copy of existing solutions.

Scoring Guidelines

  • Score 8-10 — Should be rare (top 5% quality only)
  • Score 6-7 — Strong submission
  • Score 4-5 — Average
  • Score 2-3 — Below average
  • Score 0-1 — Poor

Strict Rules Applied During Scoring

  • Vague, short, or generic answers → score LOW
  • Generic ideas ("make an app to solve X") without differentiation → Uniqueness and Creativity score 0-3
  • No explanation of why solution is better than alternatives → Uniqueness and Impact score LOW
  • No understanding of user pain points with examples → Empathy score LOW
  • No mention of adaptability or willingness to learn → Flexible Thinking MUST be 0-3
  • Remember the 60/40 primary/secondary weightage for each parameter (see next section)

6. 10 Coherence Cross-Checks

Coherence checks run in PARALLEL with main evaluation. Penalties are applied to scores after both complete.

The AI performs 10 specific logical consistency checks between pairs of questions:

#Check NameQuestions ComparedWhat It Verifies
1User-Problem FitQ1 vs Q2Does the target group match the problem described?
2Problem-Solution FitQ2 vs Q3Does the solution DIRECTLY address the problem?
3Difference ValidityQ3 vs Q4Is the claimed difference actually visible in the solution?
4Execution RealityQ3 vs Q5Are the build steps needed to create the solution?
5Resources AlignmentQ5 vs Q6Do the resources support the steps listed?
6Risk AwarenessQ6 vs Q8Do challenges reflect real constraints from resources?
7Impact ContinuityQ7 vs Q8Do challenges contradict the claimed positive impact?
8Sustainability LogicQ7 vs Q9Does the team's positioning support long-term impact?
9Team FitQ5+Q6 vs Q9Does the team have skills/resources for the steps?
10Learning LoopQ10 vs Q3+Q5Has feedback actually influenced the solution or steps?

Coherence Rules

  • Only PASS if there is a CLEAR logical connection
  • If an answer is empty or too short (<10 words), that check FAILS
  • If answers contradict each other, that check FAILS
  • If answers seem to be about different topics, that check FAILS

Coherence Penalty Mapping

Each failed check deducts -1 from specific parameters:

Disqualification Rule

If more than 5 coherence checks FAIL: ALL 10 parameter scores are set to 0. The submission is marked as DISQUALIFIED and categorized as incoherent. Final score = 0.

Before scoring, the system counts total words across all 12 question answers and tags the submission:

This tag is included in the AI prompt so the evaluator knows to penalize lazy or low-effort submissions more heavily.

8. Attachment Analysis

Students can upload 3 types of files. Attachment analysis runs in parallel with main evaluation and coherence checks.

File TypeAccepted FormatsMax SizeHow It's Analyzed
ImageJPG, JPEG, PNG, GIF, WEBP5 MBSent directly to Claude for visual analysis
DocumentPDF, DOC, DOCX, PPT, PPTX, TXT10 MBText extracted first, then sent to Claude
VideoMP4, WEBM, MOV, MPEG, MPG20 MBSent directly to Gemini 2.0 Flash for native video analysis

How Each File Type is Analyzed

Images:

  • The actual image is sent to Claude AI (base64 encoded)
  • Claude describes EXACTLY what it sees (objects, text, diagrams, scenes)
  • Claude then judges if the image DIRECTLY relates to the student's idea
  • Stock photos, random images, memes → marked IRRELEVANT

Videos:

  • The video file is sent to Gemini 2.0 Flash for native analysis (base64 encoded)
  • Gemini watches the video and describes what it shows
  • Gemini checks if the content specifically relates to the idea
  • Random footage, stock video, unrelated demos → marked IRRELEVANT
  • Fallback: If Gemini fails, frame extraction is attempted using OpenCV

Documents:

  • Text is extracted using PyPDF2 (PDF), python-docx (DOCX), python-pptx (PPTX)
  • Extracted text (up to 500 characters) is sent to Claude
  • Claude checks if the document content matches the idea

Relevance Judgment (Strict)

A file is marked IRRELEVANT if:

  • It shows generic content (stock photos, random landscapes, animals, memes)
  • It's about a DIFFERENT topic than the idea
  • It's a random screenshot unrelated to the idea
  • The connection to the idea is vague or requires stretching logic
  • It contains content about a completely different domain

Edge Cases Handled

9. Penalty System

There are two types of penalties: coherence penalties (per failed check) and attachment penalties (for missing/irrelevant files).

Coherence Penalties

Applied to individual parameters (see Section 6). Each failed check = -1 to affected parameters (minimum 0). More than 5 failures = DISQUALIFIED (all scores = 0).

Attachment Penalties

Deducted from the final score. Maximum total attachment penalty is capped at -5.

Mismatch penalty and missing types penalty add together, but are capped at -5.

10. Final Score Calculation

Step 1: AI scores 10 parameters (0-10 each)         = Raw AI Scores
Step 2: Apply coherence penalties (-1 per fail)       = Adjusted Scores
        If >5 checks fail: ALL scores = 0 (disqualified)
Step 3: Sum all 10 adjusted scores                    = Parameter Total (0-100)
Step 4: Subtract attachment penalty (capped at -5)    = Final Score

Final Score = max(0, Parameter Total - Attachment Penalty)

Final Score Range: 0 to 100

11. Re-Evaluation Logic

When an admin triggers re-evaluation for a submission:

Score Protection (Anti-Inflation)

To prevent score inflation from repeated re-evaluations, the system takes the LOWER of the old score and new score for each parameter:

Re-evaluated Score = min(Old Score, New Score)  [for each parameter]

This ensures scores can only go down or stay the same on re-evaluation, never up.

Exception: Coherence Change

Old EvalNew EvalWhat Happens
CoherentCoherentmin(old, new) for each parameter
CoherentDisqualified (>5 fails)All scores become 0
DisqualifiedCoherentNew scores are used (since old was 0)
DisqualifiedDisqualifiedAll scores remain 0

Attachment Re-Analysis

On re-evaluation, all 3 tasks (scoring, coherence, attachments) run fresh in parallel. Old evaluation is deleted before creating the new one.

12. Ranking & Top 400 System

After evaluation, submissions are ranked by final_score in descending order (higher = better).

Ranking Rules

  • Tied scores get the same rank (competition-style ranking)
  • Tiebreaker order: Uniqueness Score > Impact Score
  • Rankings update automatically after each evaluation or batch evaluation
  • Disqualified submissions are excluded from ranking (rank = null)

Top 400 Selection

A submission qualifies for Top 400 based on a single condition:

  1. Rank is within top 400 positions

There is no minimum score threshold. The best 400 ideas by score are automatically selected, regardless of their absolute score. If only 200 submissions exist, all 200 are in the Top 400. If 1000 exist, only the highest-scoring 400 qualify.

Dynamic & Flexible: Rankings are recalculated after every evaluation. If a new submission scores higher than the current 400th-ranked submission, it enters the Top 400 and pushes the previous 400th out. This means even the last student to submit can be Rank 1 if their idea is the best. The leaderboard is always live and fair.

13. Auto Title Generation

Idea titles are generated in two stages:

Stage 1: Immediate (on submission)

When a student submits their idea, the system immediately auto-generates a title from Q3 (Solution) or Q2 (Problem), truncated to ~80 characters at a word boundary. This ensures the title is visible immediately in the admin dashboard and rankings.

Stage 2: AI-Enhanced (background)

After submission, a background thread runs the AI summary processor. The AI generates a short, catchy title (5-8 words) from the full submission context. This AI-generated title overwrites the initial auto-generated title, providing a more polished display name.

Title Display Fallback Chain

Admin Dashboard:  title → q3_solution_simple → q2_exact_problem → problem_definition → 'Untitled'
Rankings:         title → q3_solution_simple → q2_exact_problem → 'Untitled'
